Basics

This application can be used to set up interactive cinematic VR experiences. A project consists of a variable number of scenes which contain interactive buttons. Each scene is associated with a video, which should be an equirectangular projection of a sphere. The video may be stereoscopic (stacked either vertically or horizontally).

Resources

In the resources section videos can be uploaded, selected for editing or deleted. Videos are stored on the application server and are not linked with the project. Due to the large size of videos it would create a significant overhead to always transfer them upon loading or saving.

Project

The project section displays the current project tree, consisting of scenes and their buttons, affordances and settings. Objects can be added or removed as necessary.

When a project is saved using "Save" from the file menu, it is stored in the browser. Use "Save As" to store it as a file on disk.

Properties

In the properties sidebar the selected item can be inspected and edited. The types of properties that are available depends on the selected item. Most items have a name which determines how they are displayed in the project tree.

Modes

In the middle section of the editor there are two buttons that change the mode of the editor. In edit mode ( mode_edit) scenes are displayed on a plane and scene elements can be manipulated. In view mode ( play_arrow) the scene is displayed in a 360-degree environment.

Scene Editor

In the scene editor (available in edit mode when a scene or one of its elements is selected) elements belonging to a scene can be manipulated.

Buttons

Buttons can be positioned by clicking and dragging them. To resize a button, hold Shift while dragging it. To select a button, double click it.

Affordances

In the context of this application an affordance is something that communicates interactivity or interaction opportunities to the user. Affordances are global to the project and affect how scenes are rendered in the viewer.

The checkbox next to an affordance in the project explorer can be used to quickly enable or disable a given affordance. This has the same effect as changing the Enabled property in the properties editor.

The order in which the affordances are applied matters. Affordances are applied top to bottom and their order can be changed by dragging the items in the list.

Types

There are various types of affordances that can be added to a project. This section lists and explains them.

The viewer has two primary parts: The video sphere and an orthographic HUD on top of that. Affordances are labeled accordingly to indicate which layers are affected. Affordances that are rendered to the sphere tend to move around as the camera changes while those on the HUD are more static.

Cursor

This renders a cursor in the middle of the screen in the shape of a ring.

Radius Factor
Radius of cursor circle as factor of the HUD width.
Line Width Factor
Line width of cursor circle as factor of the HUD width.
Opacity Default
Opacity when not hovering a button.
Opacity Hover
Opacity when hovering a button.

Shapes

This simply renders the shapes of buttons, so the user can clearly see which areas are interactive.

Fill Opacity Default
Fill opacity when not hovering the button.
Fill Opacity Hover
Fill opacity when hovering the button.
Border Thickness Factor
Border thickness of button as factor of sphere circumference.
Border Opacity Default
Border opacity when not hovering the button.
Border Opacity Hover
Border opacity when hovering the button.

Lines

This renders lines from the center of the screen to buttons.

Line Width Factor
Line width of cursor circle as factor of sphere circumference.
Line Segments
The number of line segments used to approximate the lines to the elements.
Range
How far the lines are drawn in percent (0 to 1). '1' means they are drawn completely from element to the screen center.

Edge Indicators

This renders a circle of varying radius and opacity at the viewport edge indicating off-screen elements.

Radius Factor
Radius of indicator as factor of screen width.
Radius Dyn. Factor
Factor of dynamic change of the indicator radius based on distance.
Opacity Max.
Maximum opacity of indicators based on distance.
Opacity Min.
Minimum opacity of indicators based on distance.
Trace Segments
The number of line segments used to approximate the path to the elements which is used to find the screen edge intersection point.
Draw Trace
Whether to draw the trace line.

Halos

This renders a circle of varying radius around off-screen elements.

Line Width Factor
Line width of halo circle as factor of screen width.
Radius Factor
Radius of halo circle as factor of screen width.
Samples
The number of sample points used to approximate the halo. If the halo does not wrap correctly on the horizontal seam, increasing the sample size can help.
Opacity Min.
Minimum opacity of indicators based on distance.
Trace Segments
The number of line segments used to approximate the path to the elements which is used to find the screen edge intersection point.
